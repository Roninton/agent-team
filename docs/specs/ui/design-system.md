# 设计系统规范

## 1. 目的与范围
本文档定义VS Code多AI协作平台的设计系统规范，包括设计原则、颜色体系、字体规范、组件规范、交互标准等。所有前端界面开发必须遵循本规范，确保产品体验的一致性。

## 2. 设计原则
### 2.1 一致性原则
- 所有界面元素的样式、交互、文案保持统一
- 相同功能的操作方式和反馈机制保持一致
- 遵循VS Code原生设计风格，与编辑器体验无缝融合

### 2.2 简洁性原则
- 界面布局简洁清晰，去除不必要的视觉元素
- 操作流程简化，减少用户点击次数和认知成本
- 信息层级明确，突出核心内容，弱化次要信息

### 2.3 可用性原则
- 所有交互元素有明确的状态反馈（hover/active/focus/disabled）
- 操作结果清晰可见，错误提示友好且有解决方案
- 支持键盘导航和快捷键操作，提升效率

### 2.4 响应式原则
- 界面自适应不同尺寸的编辑器窗口
- 支持不同分辨率的显示器，确保在高分屏上显示清晰
- 支持缩放调整，文字和元素不会变形或模糊

- 避免使用固定色值，所有颜色必须通过主题变量引用

## 4. 字体规范
### 4.1 字体家族
- 优先使用系统默认无衬线字体：`-apple-system, BlinkMacSystemFont, "Segoe UI", Roboto, Oxygen, Ubuntu, Cantarell, "Open Sans", "Helvetica Neue", sans-serif`
- 等宽字体使用VS Code默认编辑器字体：`var(--vscode-editor-font-family)`

### 4.2 字体大小
| 级别 | 大小（px） | 行高 | 用途 |
|------|------------|------|------|
| 大标题 | 20 | 1.5 | 页面标题、面板标题 |
| 标题 | 16 | 1.5 | 二级标题、卡片标题 |
| 正文 | 14 | 1.5 | 主要内容、消息文本 |
| 辅助文本 | 12 | 1.4 | 提示信息、时间戳、标签 |
| 小文本 | 11 | 1.3 | 角标、状态提示 |

### 4.3 字体粗细
- 常规：400，用于正文、辅助文本
- 中等：500，用于标题、强调内容
- 粗体：600，用于重要标题、高亮提示

## 5. 间距规范
### 5.1 基础间距单位
使用8px作为基础间距单位，所有间距都是8px的倍数：
- xs: 4px (0.5x)
- s: 8px (1x)
- m: 16px (2x)
- l: 24px (3x)
- xl: 32px (4x)
- xxl: 48px (6x)

### 5.2 常用间距
- 页面内边距：16px
- 元素间距：8px/16px
- 卡片内边距：16px
- 列表项内边距：8px 12px
- 按钮内边距：6px 16px（小按钮）/ 8px 20px（标准按钮）/ 10px 24px（大按钮）

## 6. 圆角规范
- 小圆角：2px，用于复选框、输入框、小标签
- 标准圆角：4px，用于按钮、卡片、弹窗
- 大圆角：8px，用于头像、提示框、大按钮
- 圆形：50%，用于头像、状态指示器

## 7. 阴影规范
| 阴影级别 | 值 | 用途 |
|----------|----|------|
| 一级阴影 | 0 1px 2px rgba(0, 0, 0, 0.1) | 卡片、按钮悬浮 |
| 二级阴影 | 0 2px 8px rgba(0, 0, 0, 0.15) | 下拉菜单、弹窗 |
| 三级阴影 | 0 4px 16px rgba(0, 0, 0, 0.2) | 模态框、拖拽元素 |

## 8. 基础组件规范
### 8.1 按钮
#### 类型
- 主按钮：蓝色背景，白色文字，用于主要操作
- 次按钮：透明背景，边框，用于次要操作
- 文字按钮：只有文字，无背景无边框，用于轻量操作
- 危险按钮：红色背景，用于删除等危险操作
- 禁用按钮：透明度0.4，不可点击

#### 状态
- 默认：正常状态
- Hover：背景色变浅/变深，有阴影
- Active：背景色更深，有按压效果
- Disabled：半透明，不可点击
- Loading：显示加载动画，文字置灰

#### 尺寸
- 大按钮：高度36px，字体14px
- 标准按钮：高度28px，字体14px
- 小按钮：高度22px，字体12px

### 8.2 输入框
#### 类型
- 单行输入框：用于短文本输入
- 多行输入框：用于长文本输入
- 搜索输入框：带搜索图标和清除按钮
- 密码输入框：带密码显示切换按钮

#### 状态
- 默认：正常状态，有边框
- Focus：边框变色，有高亮效果
- Hover：边框颜色加深
- Disabled：背景变灰，不可输入
- 错误状态：边框变红，底部显示错误提示

### 8.3 复选框/单选框
- 大小：16px x 16px
- 选中状态：蓝色背景，白色对勾/圆点
- 未选中状态：白色背景，灰色边框
- 禁用状态：半透明，不可点击

### 8.4 下拉选择器
- 高度：28px，与标准按钮一致
- 下拉面板最大高度：300px，超出滚动
- 选项高度：28px，内边距8px 12px
- 选中选项：蓝色背景，白色文字
- Hover选项：浅灰背景

### 8.5 标签
- 高度：20px，内边距0 8px，圆角2px
- 字体大小：12px
- 类型：普通标签（灰底灰字）、成功标签（绿底白字）、警告标签（黄底白字）、错误标签（红底白字）、信息标签（蓝底白字）

### 8.6 消息提示（Toast）
- 位置：编辑器底部居中
- 最小宽度：300px，最大宽度：600px
- 内边距：12px 16px
- 圆角：4px，阴影：二级阴影
- 自动消失时间：3秒（成功/信息）/ 5秒（警告/错误）
- 类型：成功、错误、警告、信息

### 8.7 模态框
- 宽度：400px（小）/ 600px（中）/ 800px（大）
- 头部：高度48px，内边距0 16px，底部边框
- 内容区：内边距16px，最小高度60px
- 底部：高度56px，内边距0 16px，顶部边框，按钮右对齐
- 遮罩层：黑色背景，透明度0.5
- 关闭按钮：右上角，24px x 24px

### 8.8 卡片
- 内边距：16px
- 圆角：4px
- 边框：1px（浅色主题#CECECE，深色主题#3E3E42）
- 阴影：一级阴影（悬浮时）
- 间距：卡片之间间距16px

### 8.9 列表
- 列表项高度：28px/32px/36px
- 内边距：8px 12px
- 分割线：1px边框，颜色与边框色一致
- Hover状态：背景色变为悬浮背景色
- 选中状态：背景色变为选中背景色，左侧蓝色边框

## 9. 图标规范
### 9.1 图标库
- 使用VS Code内置图标库（Codicons）作为主要图标源
- 图标名称遵循Codicons命名规范：`$(icon-name)`
- 特殊场景可以使用自定义SVG图标，但风格必须与Codicons保持一致

### 9.2 图标尺寸
- 小图标：12px x 12px，用于小标签、角标
- 标准图标：16px x 16px，用于按钮、菜单、列表项
- 大图标：24px x 24px，用于面板标题、空状态
- 超大图标：32px x 32px，用于欢迎页、错误页

### 9.3 图标颜色
- 默认与同级别文字颜色一致
- 禁用状态使用三级文本颜色
- 特殊状态使用对应的辅助色（成功/警告/错误/信息）

## 10. 交互规范
### 10.1 鼠标交互
- 可点击元素：鼠标悬停时变为手型指针
- 不可点击元素：鼠标悬停时变为默认指针
- 拖拽元素：鼠标悬停时变为移动指针
- 文本区域：鼠标悬停时变为文本指针

### 10.2 反馈机制
- 所有操作必须有即时反馈：按钮点击效果、操作成功提示、失败错误提示
- 耗时操作必须显示加载状态：按钮loading、进度条、骨架屏
- 操作结果必须清晰告知用户：成功/失败原因、下一步操作建议

### 10.3 快捷键
- 常用操作必须支持快捷键：发送消息(Ctrl+Enter)、新建会话(Ctrl+N)、搜索(Ctrl+F)
- 快捷键提示：在按钮tooltip、菜单中显示对应的快捷键
- 支持自定义快捷键：用户可以在设置中修改快捷键配置

### 10.4 动画效果
- 过渡动画：所有状态切换使用0.2s的ease过渡效果
- 加载动画：使用VS Code原生的加载动画样式
- 避免过度动画：动画效果简洁，不影响操作流畅度
- 性能优先：动画使用CSS transform和opacity属性，保证60fps流畅度

## 11. 无障碍规范
- 所有可交互元素必须有合适的aria-label属性
- 图片、图标必须有alt文本描述
- 支持键盘导航：所有操作可以通过Tab键切换，Enter键触发
- 颜色对比度符合WCAG 2.1 AA标准：文本与背景对比度≥4.5:1
- 支持屏幕阅读器：所有界面元素可以被屏幕阅读器正确识别和朗读
