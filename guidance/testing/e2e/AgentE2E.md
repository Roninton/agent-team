## 📋 执行流程

### 第一步：环境准备（测试数据完全隔离，不影响正常用户）
1. 打开终端，进入项目根目录 `/home/hyh/project/agent-team`
2. **🔴🔴🔴 最高优先级强制要求：启动测试专属服务（必须使用独立数据目录，绝对不能省略）**：
   ```bash
   # 必须加这个环境变量！指定测试数据存储目录为 ./data-test，完全独立于默认的 ./data 目录
   ACP_DATA_ROOT=./data-test npm run dev
   ```
   ❌ 绝对不能直接运行 `npm run dev`，否则会污染正常用户的数据，这是最高优先级规则，比其他所有规则都重要！
   ✅ 加了这个变量后，测试产生的所有数据库、日志、代理数据都会存在`./data-test`目录，完全不会影响正常用户的数据
3. 等待服务完全启动：后端服务在3001端口，前端服务在3000端口
4. 打开浏览器，访问 `http://localhost:3000`，验证页面可以正常加载
5. 确保系统处于初始状态，无残留测试数据

### 第二步：测试执行规则
你需要**按顺序**执行以下目录下的所有测试用例：
1. `01-smoke-tests.md` - 冒烟测试（P0，必须全部通过才能继续）
2. `02-agent-tests.md` - 代理管理测试（P0，核心功能）
3. `03-chat-tests.md` - 聊天功能测试（P0，核心功能）

#### 执行要求：
1. **完全模拟真实用户操作**：所有操作都通过浏览器页面手动点击、输入完成，不要调用API或修改数据库
2. **🔴 严格低耦合独立执行（最高优先级）：
   - 每个用例执行前优先通过页面操作清理测试产生的数据（删除测试代理、删除测试会话等）
   - 只有必要时（页面状态被严重污染）才删除localStorage/cookie，尽量模拟真实用户正常使用场景
   - 测试全程使用独立的测试数据目录，不要使用默认用户数据位置，不影响正常用户的数据
   - 绝对不要依赖前一个用例的执行结果和留下的数据
   - 每个用例都可以单独运行，不受其他用例影响
3. **严格按步骤执行**：完全按照测试用例描述的操作步骤执行，不要省略任何步骤
4. **完整记录证据**：
   - 每个用例执行完成后，截图整个浏览器页面（包含地址栏、页面内容、控制台）
   - 失败用例需要额外截图错误信息、控制台日志、网络请求详情
   - 所有截图按`测试类别/用例ID_状态.png`格式命名保存
5. **结果更新**：每个用例执行完成后，直接在对应的测试用例.md文件中更新测试结果：
   - 测试通过：把 `⬜ 未测试` 改为 `✅ 通过`
   - 测试失败：把 `⬜ 未测试` 改为 `❌ 失败`，并在后面补充失败原因和截图链接

### 第三步：异常处理规则
1. **单个用例失败**：不要停止测试，记录失败原因后继续执行后面的用例
2. **系统崩溃/服务停止**：尝试重新启动服务，恢复后继续执行未完成的测试
3. **用例描述不清晰**：按照最符合用户操作习惯的方式执行，记录你的理解和执行过程
4. **遇到无法解决的阻塞问题**：记录问题详情，截图保存，跳过该用例继续执行

## 📌 执行原则
1. **事实优先**：所有结果必须基于实际执行情况，不要猜测或脑补
2. **客观中立**：如实记录所有问题，不要隐瞒或美化
3. **用户视角**：站在普通用户的角度评估功能是否符合预期
4. **完整严谨**：所有操作和结果都要有证据支持，不要遗漏任何细节
