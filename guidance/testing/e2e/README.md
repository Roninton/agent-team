# Agent 驱动 E2E 回归测试指南

本指南用于指导基于AI Agent的End-to-End回归测试执行，充分利用AI能力简化测试流程，降低维护成本。

> 📌 **方案调整说明**：原基于Playwright的自动化E2E测试方案已撤销，改为Agent驱动的手动回归测试方案，解决了原方案环境配置复杂、维护成本高、稳定性差等问题。

---

## 测试方案核心优势

| 维度 | 原Playwright自动化测试 | 新Agent驱动测试 |
|------|-----------------------|----------------|
| **环境配置** | 复杂，需适配浏览器、依赖版本 | 简单，仅需启动`npm run dev` |
| **维护成本** | 高，UI变更即需要更新测试用例 | 低，用例自然语言描述，自动适配UI变化 |
| **稳定性** | 低，受网络、时序、状态等变量影响 | 高，Agent智能识别和容错 |
| **覆盖能力** | 有限，仅覆盖预定义场景 | 强，可探索性测试，发现未知问题 |
| **问题定位** | 困难，仅能提供错误日志 | 简单，自动截图、记录重现步骤 |

## 快速开始

### 1. 测试准备
- 启动前后端服务：`npm run dev`
- 系统状态重置，初始化测试数据
- 打开浏览器页面，准备测试执行

### 2. 执行冒烟测试（优先级最高）
目标：确保系统基本可用
- 服务正常启动，API健康检查通过
- 前端页面正常加载，无控制台错误
- 核心功能基础操作正常

### 3. 执行全量功能测试
目标：覆盖所有核心功能场景
- 页面功能测试（6个用例）
- Agent功能测试（11个用例）
- 多Agent交互测试（8个用例）
- 交叉功能测试（7个用例）
- 数据持久化测试（8个用例）

### 4. 执行边界场景测试
目标：验证系统异常处理和鲁棒性
- 网络异常、服务崩溃恢复
- 高并发、限流场景
- 边界条件、异常输入

## Agent 测试执行原则

1. **完全模拟真实用户行为**
   - 用真实的点击、输入、等待操作
   - 按照用户真实使用流程执行测试
   - 不跳过任何正常用户会经历的步骤

2. **用例独立执行**
   - 每个用例从零开始执行
   - 不依赖前一个用例的执行结果
   - 测试前重置系统到初始状态

3. **完整记录测试证据**
   - 每个用例执行后截图保存
   - 失败用例详细记录重现步骤
   - 控制台错误、网络请求日志完整保存

4. **探索性测试补充**
   - 完成预定义用例后，进行自由探索测试
   - 尝试非常规操作路径，发现潜在问题
   - 记录所有发现的问题，不管是否在预期内
