# 冒烟测试用例

> 🎯 目标：验证系统基本可用，每次构建后必须 100% 通过
>
> ⏱️ 执行时间：< 5 分钟

---

## 测试前置条件

1. 后端服务已启动（端口 3001）
2. 前端服务已启动（端口 5173）
3. 浏览器可以正常访问两个端口

---

## 📋 测试用例列表

### Test S001: 服务启动检查

| 项 | 内容 |
|----|------|
| **优先级** | P0 - 阻塞 |
| **前置** | 无 |
| **测试步骤** | |
| 1 | 访问 `http://localhost:3001/api` |
| 2 | 访问 `http://localhost:5173` |
| **预期结果** | |
| 1 | API 返回 200（可以是 404，只要服务响应就行） |
| 2 | 前端页面正常加载，没有白屏 |
| **备注** | 这是所有测试的基础，不通过后续测试不用跑 |

---

### Test S002: 代理列表正常显示

| 项 | 内容 |
|----|------|
| **优先级** | P0 |
| **前置** | Test S001 通过 |
| **测试步骤** | |

| 2 | 查看代理列表 |

### Test S004: 发送消息功能正常

| 项 | 内容 |

### Test S006: WebSocket 连接正常

## ✅ 冒烟测试通过标准

- 以上 6 个用例 **全部通过**，没有失败
- 整个执行过程 **没有出现 JavaScript 报错**
- 所有操作的响应时间都在 **可接受范围**（一般 < 3 秒）

---

## 🔄 失败处理流程

1. 冒烟测试失败 → 立即停止后续测试
2. 检查最近的代码变更
3. 优先修复冒烟测试失败的问题
4. 修复后重新执行完整的冒烟测试
