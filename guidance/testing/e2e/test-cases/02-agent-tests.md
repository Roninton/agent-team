# 🧪 代理管理功能测试用例
> 优先级：P0，覆盖代理CRUD、生命周期管理所有核心功能

---

#### 用例ID: AGENT-001
#### 测试场景: 可以正常添加新代理
#### 前置条件: 
- 前后端服务已正常启动
- 系统已重置到初始状态
- 已打开设置页面`http://localhost:3000/settings`
#### 操作步骤:
1. 点击"添加代理"按钮，打开新增代理表单
2. 填写代理信息：
   - 名称：随机生成的测试代理名称（例如：test-agent-xxx）
   - 类型：选择"自定义命令"
   - 命令：填写`python -c "while True: import sys; print(f'ECHO: {input()}')"`
   - 其他配置保持默认
3. 点击"保存"按钮提交表单
4. 查看代理列表是否显示新添加的代理
#### 验收标准:
- ✅ 表单提交成功，无错误提示
- ✅ 新添加的代理出现在代理列表中
- ✅ 代理状态显示为"运行中"（添加后默认自动启动）
- ✅ 无控制台错误
#### 测试结果: ✅ 通过
- 验证细节：新代理test-agent-echo成功添加到列表，配置信息显示正确，无控制台功能错误，仅有一个antd组件警告不影响使用
