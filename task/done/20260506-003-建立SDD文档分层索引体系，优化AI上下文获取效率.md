# 建立SDD文档分层索引体系，优化AI上下文获取效率

## 描述
当前SDD文档分散在各个目录，没有统一的索引结构，导致AI在开发过程中难以快速、完整地获取相关上下文。需要建立分层次、无冗余的SDD索引体系，同时明确AI获取上下文的规则，支持递归式快速获取从顶层架构到具体功能的全链路文档。

## 需求要点
1. 建立分层级的文档索引结构，支持从顶层到底层的快速关联
2. 定义文档引用规则，避免内容冗余
3. 明确AI开发时的上下文获取顺序和方法
4. 索引结构要支持快速定位相关的架构、接口、规范文档

## 验收标准
1. [x] 完成`docs/SPEC_INDEX.md`分层索引文档编写，按架构、功能、协议、UI等维度分类索引所有SDD文档
2. [x] 定义文档引用规范，明确跨文档引用的格式和规则：已写入索引文档和instruction.md
3. [x] 更新`.copilot/instruction.md`，新增AI上下文获取规则，明确开发时的文档读取顺序：已新增专门的上下文获取规则章节
4. [x] 索引体系支持快速检索任意功能对应的全链路相关文档：分层结构已支持从顶层到底层的全链路关联
5. [x] 所有现有SDD文档已加入索引：已将所有已编写的规范文档加入对应层级的索引
6. [x] 新增任务创建规范：明确要求创建任务时必须提前获取上下文，将核心摘要记录到任务文档中，避免反复读取
