### 新增需求清单
1. **群聊消息限流与静默策略**：
   - 支持Agent可选回复机制，不需要对每条消息都生成回复，防止消息爆炸
   - 兜底限流策略：单个Agent每分钟最多发送5条消息

2. **Agent独立配置与可视化编辑**：
   - 用户点击Agent头像可直接修改Agent的`agent.md`初始化文件
   - 支持每个Agent独立设置专属的`instruction.md`指令，构造上下文时自动嵌入

3. **持久化方案调整**：
   - Agent配置文件使用文件系统存储（适配非结构化、变长内容）
   - 会话、消息、任务状态使用Redis做缓存，可选持久化到轻量数据库
   - 移除不必要的重型数据库依赖

4. **上下文自动裁剪与解包**：
   - 构造上下文时自动过滤无关内容，避免冗余消息传输
   - 消息与上下文支持自动解包，按需加载，避免一次性传入过多无关内容
