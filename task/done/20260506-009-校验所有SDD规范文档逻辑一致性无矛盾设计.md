# 校验所有SDD规范文档逻辑一致性与无矛盾设计

## 核心上下文摘要
### 校验范围
覆盖所有已完成的SDD规范文档，共16篇：
1. **Level 0顶层规范**：
   - `.copilot/instruction.md`（开发元规则、工具调用规范）
   - `docs/DEVELOPMENT_GUIDELINE.md`（SDD开发规范总纲）
   - `docs/specs/architecture/tech-stack-and-architecture.md`（整体技术栈与架构规范）
   - `docs/SPEC_INDEX.md`（SDD索引总表）

2. **Level 1架构层规范**：
   - `docs/specs/architecture/group-management.md`（群组管理子系统）
   - `docs/specs/architecture/agent-scheduling.md`（代理调度子系统）
   - `docs/specs/architecture/task-allocation.md`（任务分配子系统）
   - `docs/specs/architecture/message-routing.md`（消息路由子系统）
   - `docs/specs/architecture/backend-architecture.md`（后端服务整体架构）

3. **Level 2功能层规范**：
   - `docs/specs/features/multi-agent-connection.md`（多代理并行连接功能）
   - `docs/specs/features/group-chat.md`（群聊功能）
   - `docs/specs/features/context-sync.md`（上下文同步功能）

4. **Level 3协议层规范**：
   - `docs/specs/protocol/acp-extension.md`（ACP协议扩展）
   - `docs/api/rest-api.md`（REST API规范）
   - `docs/api/websocket-api.md`（WebSocket消息协议）

5. **Level 4 UI层规范**：
   - `docs/specs/ui/design-system.md`（设计系统规范）
   - `docs/specs/ui/chat-interface.md`（群聊界面规范）

### 校验维度
1. **跨文档一致性校验**：相同的定义、规则在不同文档中是否一致，没有矛盾
2. **架构逻辑一致性校验**：上下层依赖关系、模块交互逻辑是否自洽，无设计冲突
3. **接口一致性校验**：API定义、消息格式在相关文档中是否统一
4. **命名一致性校验**：所有实体、字段、概念的命名在所有文档中统一
5. **规则一致性校验**：限流策略、权限规则、存储规则等在相关文档中无矛盾
6. **索引一致性校验**：`SPEC_INDEX.md`中的文档列表与实际文件是否完全匹配，描述一致

## 验收标准
1. [x] 所有文档中相同概念、定义、命名完全统一，无冲突
