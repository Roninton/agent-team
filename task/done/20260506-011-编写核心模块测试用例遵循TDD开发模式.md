# 编写核心模块测试用例，遵循TDD开发模式

## 核心上下文摘要
### 开发模式
遵循TDD（测试驱动开发）模式：先写测试用例 → 运行测试看失败 → 实现功能 → 测试通过 → 重构优化。

### 优先编写测试的核心模块
1. **Agent模块**：代理生命周期管理、配置管理、限流控制
2. **Session模块**：会话管理、连接管理
3. **Message模块**：消息路由、限流校验
4. **Context模块**：上下文裁剪、增量同步

### 测试框架
使用VS Code扩展官方推荐的测试框架，配合mocha、chai断言库，测试用例放在各模块的`__test__`目录下。
