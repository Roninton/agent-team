# 后端模块代码迁移，将旧src代码迁移到NestJS架构

## 任务概述
将原src目录中的各模块代码迁移到server目录的NestJS模块化架构中，重构为符合NestJS规范的Module、Service、Controller结构。

## 上下文信息
- 原代码位置：`/home/hyh/project/vscode-acp/src/` 下的各模块
- 目标位置：`/home/hyh/project/vscode-acp/server/src/modules/`
- 框架：NestJS，使用Module、Service、Controller、Gateway模式
- 测试框架：Jest（NestJS默认）

## 重要架构说明
原代码大量依赖VS Code API（vscode.workspace, vscode.window等），在独立NestJS应用中需要重构：

### VS Code依赖处理策略
1. **FileSystemHandler** → 使用Node.js原生fs模块替代VS Code fs API
2. **PermissionHandler** → 实现Web版权限确认API（前端弹窗确认）
3. **TerminalHandler** → 使用Node.js child_process + WebSocket实时输出
4. **Logger** → 使用NestJS内置Logger替代

### 模块依赖关系
```
AgentService (无VS Code依赖)
    ↓
MessageRouter (依赖AgentService)
    ↓
GroupManager (依赖MessageRouter + AgentService)
    ↓
Session/Connection (依赖AgentService + Handlers)
    ↓
Protocol/ACP (依赖所有)
```

## 迁移规范

### NestJS模块结构标准
```
modules/agent/
├── agent.module.ts          # 模块定义
├── agent.service.ts         # 业务逻辑
├── agent.controller.ts      # REST API
├── agent.gateway.ts         # WebSocket (如需)
├── types/
│   └── agent.types.ts       # 类型定义
├── dto/
│   ├── create-agent.dto.ts  # 数据传输对象
│   └── update-agent.dto.ts
└── __test__/
    └── agent.service.spec.ts
```

### 代码重构原则
1. 使用NestJS依赖注入替代直接实例化
2. 使用TypeORM实体替代纯数据结构
3. 使用NestJS Logger替代自定义Logger
4. 使用@nestjs/websockets实现WebSocket网关
5. 使用class-validator进行DTO验证
6. 保持原有业务逻辑不变，仅调整架构
