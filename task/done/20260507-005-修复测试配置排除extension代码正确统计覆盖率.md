# 任务ID: 20260507-005
## 任务名称：修复测试配置，排除extension历史代码，正确统计覆盖率

## 任务概述
当前测试覆盖率统计包含了旧的extension历史代码，且新的NestJS模块测试虽然存在但未正确统计（显示0%）。需要修复vitest配置，排除extension目录，确保只统计主项目代码的覆盖率。

## 上下文信息

### 相关规范文档索引
**Level 0 顶层规范：**
- `.copilot/instruction.md` - SDD→TDD流程规范
- `docs/DEVELOPMENT_GUIDELINE.md` - 开发规范总纲

**当前问题：**
1. `src/extension/` 是历史遗留代码，不应纳入主项目覆盖率统计
2. 新的NestJS模块（agent、group等）虽然有 `*.spec.ts` 文件，但覆盖率显示0%
3. 测试文件命名不统一，既有 `*.test.ts` 也有 `*.spec.ts`

### 技术约束
- 使用 vitest 作为测试框架
- 目标：行覆盖率 ≥ 80%
- 只统计 `src/server/src/modules/**` 和 `src/client/src/**` 代码
- 排除 `src/extension/**` 所有代码

## 执行计划

### 阶段一：修复 vitest 配置
1. 更新 `vitest.config.ts`
   - 添加 `src/extension/**` 到 exclude 列表
   - 明确 include 路径为 `src/server/src/**/__test__/**/*.{test,spec}.ts`
   - 明确 coverage include 路径为 `src/server/src/**/*.ts`
   - 排除 `src/server/src/**/*.d.ts`, `src/server/src/**/main.ts`

2. 验证配置生效
   - 运行 `npm run test:coverage`
   - 确认 extension 代码不再出现在报告中
   - 确认 NestJS 模块测试正确统计

### 阶段二：统一测试文件命名规范
1. 所有测试文件统一使用 `*.spec.ts` 后缀（NestJS 规范）
   - `AgentManager.test.ts` → `agent-manager.spec.ts`
   - 其他测试文件同理

2. 按模块组织测试目录
   - 每个模块的 `__test__/` 目录下：
     - `*.controller.spec.ts` - Controller 层测试
     - `*.service.spec.ts` - Service 层测试
     - `*.integration.spec.ts` - 集成测试（可选）

### 阶段三：验证修复效果
1. 运行完整测试套件
2. 确认覆盖率报告正确
3. 确认所有测试通过

## 验收标准
1. ✅ `src/extension/` 代码完全排除在覆盖率统计之外
2. ✅ NestJS 6个模块的测试正确执行并统计覆盖率
3. ✅ 测试文件命名统一为 `*.spec.ts`
4. ✅ 覆盖率报告不再有0%的模块（除了尚未实现的部分）
5. ✅ 所有测试全部通过
