### 测试编写规范
#### 1. 测试文件结构
```typescript
import { render, screen, waitFor } from '@testing-library/react';
import userEvent from '@testing-library/user-event';
import { ChatPage } from '../ChatPage';

// Mock API 调用
vi.mock('../api/agent.api', () => ({
  getAgents: vi.fn(),
  sendToAgent: vi.fn(),
}));

describe('ChatPage', () => {
  beforeEach(() => {
    vi.clearAllMocks();
  });

  it('should render chat page correctly', () => {
    // Arrange & Act
    render(<ChatPage />);
    
    // Assert
    expect(screen.getByRole('main')).toBeInTheDocument();
    expect(screen.getByRole('textbox')).toBeInTheDocument();
  });

  it('should send message when user types and clicks send', async () => {
    // Arrange
    const user = userEvent.setup();
    render(<ChatPage />);
    
    // Act
    await user.type(screen.getByRole('textbox'), 'Hello, world!');
    await user.click(screen.getByRole('button', { name: /send/i }));
    
    // Assert
    await waitFor(() => {
      expect(screen.getByText('Hello, world!')).toBeInTheDocument();
    });
  });
});
```

#### 2. 测试最佳实践
- ✅ 使用 `@testing-library/user-event` 模拟用户交互
- ✅ 使用语义化查询（role、labelText、placeholderText）
- ✅ 避免测试实现细节，测试用户行为
- ✅ 异步操作使用 `waitFor` 或 `findBy*` 查询
- ✅ 每个测试用例独立，不共享状态
- ✅ 测试文件与组件文件同目录或 `__test__` 子目录
