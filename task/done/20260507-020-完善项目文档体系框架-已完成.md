#### 3.2 test-cases/ 测试用例

##### 01-smoke-tests.md (冒烟测试)
目标：验证系统基本可用，每个核心功能至少走一遍
```
测试用例列表：
1. 服务启动测试
   - 后端服务正常启动（3001端口）
   - 前端页面正常加载（3000端口）
   - API 健康检查返回 200

2. 代理生命周期测试
   - 查看默认代理列表（3个默认代理存在）
   - 启动代理（状态变为 running）
   - 停止代理（状态变为 stopped）

3. 群聊基本功能
   - 发送消息
   - 接收消息
   - 消息显示正确

4. 群组管理基本操作
   - 查看默认群组（3个群组存在）
   - 创建新群组
   - 删除群组
```

##### 02-integration-tests.md (集成测试)
目标：验证多模块协作，覆盖单元测试边界
```
测试用例列表：
1. 多代理并发场景
   - 同时启动2个代理
   - 分别发送消息
   - 两个代理的回复正确路由到对应会话

2. 完整消息流测试
   前端发送消息 → REST API → MessageService → AgentManager →
   ACP 协议 → 子进程处理 → 回复 → WebSocket → 前端显示

3. 群组-代理关联测试
   - 创建包含多个代理的群组
   - 在群聊中 @ 特定代理
   - 验证只有被 @ 的代理回复

4. 异常场景恢复
   - 代理进程异常退出
   - 系统检测到状态变化
   - 用户可以重新启动
```

##### 03-edge-cases.md (边界/压力场景)
目标：验证系统鲁棒性
```
测试用例列表：
1. 压力边界
   - 连续发送100条消息（限流生效）
   - 同时启动10个代理（达到上限）

2. 长消息/大附件
   - 发送超长文本消息（>10000字）
   - 消息显示正常不溢出

3. 并发操作
   - 多标签页同时操作同一个代理
   - 状态最终一致

4. 网络异常
   - WebSocket 断开重连
   - 断线期间消息不丢失
```
